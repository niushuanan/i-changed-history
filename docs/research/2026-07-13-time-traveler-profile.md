# 时空人格改造：GitHub 源码调研与落地设计

## 问题定义

旧版“名字、职业、能力、风险偏好”是资料收集，不是游戏机制。玩家不知道填写后会改变什么；表单超过一屏，入口还会因为容器裁切而无法继续。新版本必须满足三个条件：一屏完成当前操作、每个选择都改变后续玩法、玩家在进入历史前就能看懂人格的实际收益和代价。

## 新机制：四次历史困境校准

玩家不再填写名字和职业。入口依次给出四个真实感困境，每次只选择一种本能：

1. `I / E`：危险情报到手后，独自验证还是先组织盟友。
2. `S / N`：史书与现场矛盾时，相信可核实证据还是追踪隐藏模式。
3. `T / F`：制度与具体的人冲突时，优先结构后果还是具体生命。
4. `J / P`：计划失效时，重建秩序还是临场变招。

四次选择得到一个时空人格代码，例如 `INTP`，并映射为一个中文原型与能力。它不是心理学诊断，只是这局游戏的规则生成器。

## 人格必须产生的三项玩法差异

1. **专属行动**：每一幕三个 AI 行动中，恰好一个按照四维人格生成并标记。
2. **因果预判**：玩家每幕可揭示一次专属行动的隐藏信息；不同人格优先看见不同信息，例如制度后果、受益者、即时证据或长期连锁。
3. **自由改命**：三次自由输入仍受当前身份、资源和历史约束，但 AI 必须根据人格能力给出一种可执行优势，同时明确代价。

## 入口与配乐

- 入口是固定高度的单幕流程，不发生页面滚动。
- 第一次点击“开始校准”属于用户手势，同时启动配乐；随后选卡、等待、事件和结局连续播放。
- 顶部始终保留静音按钮；浏览器拒绝播放时不阻断游戏。
- 结果页用三条短句直接说明人格在游戏中的作用，再进入五十个历史瞬间。

## 数据兼容

- 存档升级到 v6，新增 `typeCode` 和四维 `dimensions`。
- v5 进行中的职业档案会确定性映射到四维人格，不丢失已经开始的十二幕；v4 正在生成和 v5 正在裁决的状态均有迁移回归测试。
- 旧字段暂时保留为内部兼容字段，用于历史卡排序与本地保底文本；用户界面不再展示这些陈旧问题。

## 验收标准

- 390 x 844 与较矮视口均能完成四次校准和进入选卡，无纵向滚动。
- 开始校准后调用一次音频 `play()`，进入选卡前音乐已经启动。
- 每种结果都有四位代码、中文原型、专属行动说明、预判视角和自由改命规则。
- DeepSeek 的开场、续写和自由改命 prompt 均包含结构化人格与能力规则。
- 自由改命 JSON 必须返回 `personalityLeverage` 并包含本局四位代码；结构修复与本地保底也必须保留人格影响。
- v5 存档可迁移，刷新后人格和进行中的游戏保持不变。
