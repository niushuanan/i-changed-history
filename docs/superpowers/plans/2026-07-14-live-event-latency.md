# 现场扫码版生成性能实施计划

1. **锁定基线与失败测试**
   - 为自由改写同步入正史、无裁决请求、快/高推理请求体、SSE 指标、部分草稿、无 `rippleLens` 合同新增失败测试。
   - 扩展 soak 报告，输出请求数、首可读时间、总时长、修复率、缓存命中和正史连续性。

2. **收紧领域合同**
   - 从 `TimelineTurn`、固定开场、提示与测试夹具删除运行时无用的 `rippleLens`。
   - 将自由改写改为 reducer 同步提交，删除 `custom-action` 请求意图和运行时裁决依赖。
   - 保证原文、传播媒介、直接结果和最高影响级别一次性进入世界正史。

3. **实现可观测的快路径**
   - 给 DeepSeek 传输增加 `fast/high` 模式、请求类型、首 reasoning/content 时间、完整时间、usage/cache 指标和限频部分 JSON 回调。
   - 主续幕走 fast；字段补丁走 fast；完整恢复与双报告走 high。
   - 保留 8192 token 上限、90 秒超时、SSE 和网络重试。

4. **实现流式草稿体验**
   - 在 hook 中保存仅内存的 generation draft。
   - 等待页展示已闭合的标题、现场和目标，完整校验前不显示可点击选项。
   - 增加 reduced-motion 与 390 x 844 布局验证。

5. **强化玩家正史可见性**
   - 在续幕紧凑展示最近一条仍在生效的玩家原文，不增加独立卡片。
   - 提示和校验保证后三幕账本逐字存在，结局人物年表逐字存在，AI 后果不得否定。

6. **真实 A/B 与十局验收**
   - 先跑代表性快/高推理样本，人工审阅历史锚点、行动具体性、因果和惊奇度。
   - 跑十个不同开局、每局四至五次唯一自由改写的生产链路。
   - 对照规格中的时延、稳定性、修复率和正史指标；未达标则基于日志继续调整。

7. **浏览器与发布收尾**
   - 跑全量单测、类型检查、构建、portable 检查和 `git diff --check`。
   - 在 390 x 844 实际点击选卡、普通决定、自由改写、流式等待、下一幕、退出。
   - 更新 `AGENTS.md` 与 `PROJECT_CONTEXT.md`，提交并推送 `main`。
