# 单一历史与双页结局设计

## 目标

让玩家只关注一件事：十二次亲手决定如何改变一个人的一生，并最终改变 2026。删除人格测试、人格行动和隐藏自动时间线，避免系统替玩家做决定。

## 游戏路径

1. 首屏直接展示按时间排列的 50 个历史瞬间。
2. 玩家选择入口，DeepSeek 生成同一主角的一生。
3. 每幕展示一个历史现场、一个因果回执、三个具体行动和一次可选的直接改写入口。
4. 十二次决定全部进入唯一的玩家正史；第十二次决定后主角死亡。
5. 一次结局请求生成两页报告：穿越者列传与被改变的 2026。

## 移动端布局

事件页在 390 x 844 内使用单一纵向网格：52px 进度页头、164px 场景图、剩余空间为弹性正文。页头不覆盖图片；三个行动平分剩余决策区；自由改写固定在底部，页面不滚动且不留大块空白。

## 结局

- `穿越者列传`：十二项年表、白话人物列传、文言人物列传，允许纵向阅读。
- `被改变的 2026`：四段身后历史、三个普通人生活细节和小说式尾声，必须在一个手机页面内读完。

两页来自同一条玩家正史，不包含性格标签、人格对照或另一条自动人生。

## 稳定性

状态机、请求和存档都只有一条历史线。存档升级到 v11，旧人格存档安全回到历史胶片。DeepSeek 仍使用 8192 token 上限；Zod 校验白话、文言、十二节点和四段身后历史的完整性。
