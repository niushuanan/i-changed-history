# 固定首幕设计

## 目标

50 张历史卡片各自拥有可立即进入的固定第一幕。玩家选卡后不等待、也不调用 DeepSeek；完成第一幕决定后，第 2 至第 12 幕继续由 DeepSeek 实时推演，结局仍由两路模型并发生成。

## 架构

- `src/data/fixedOpenings.ts` 是固定首幕的唯一数据入口，按历史种子 ID 返回通过 `TimelineTurn` schema 校验的第一幕。
- 首幕从历史种子的日期、地点、身份、三条史实和真实结局构建；三个行动固定覆盖微调、改制和断裂，并在代码库中稳定复现。
- `START_SCENARIO` 在 reducer 内同步装载首幕并直接进入 `event`，不再创建 `opening` 请求。
- DeepSeek 只保留续幕、自由改写影响分类、人物列传和 2026 世界报告。
- `generationSource` 区分 `fixed` 与 `deepseek`，页面如实展示来源。

## 稳定性与兼容

- 所有 50 个固定首幕在测试中逐个通过 schema、年份、地点、身份、三选项和来源校验。
- 删除运行时开场请求后，旧存档中的开场生成状态在读取时回到历史胶片，不继续发送已废弃请求。
- README 明确模型调用边界、环境变量、Windows/macOS/Linux 启动和完整验证命令。

## 验收

1. 点击任意卡片立即进入第一幕，不出现等待页。
2. 第一幕不产生 DeepSeek 请求，页面显示“固定历史开场”。
3. 提交第一幕决定后仍请求 DeepSeek 生成第二幕。
4. 50 个首幕全部可解析、可选择、可进入后续状态。
5. 测试、类型检查、生产构建和跨平台扫描全部通过。
