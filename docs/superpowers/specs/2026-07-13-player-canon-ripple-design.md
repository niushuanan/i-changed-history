# 玩家正史与因果涟漪改造规格

## 目标

解决三个直接影响乐趣的问题：

1. 自由输入被当作“申请行动”，模型会擅自判失败，玩家没有真正改写历史。
2. 后续章节沿着上一事件做线性续集，缺少跨人群、跨制度和跨生活领域的蝴蝶效应。
3. 时空人格每题只有两个答案，像简化问卷，不像游戏中的困境选择。

## 方案比较

### 方案 A：继续做历史可行性裁判

模型根据身份、资源和时代判断行动成功或失败。优点是写实，缺点是玩家最重要的自由输入经常被否定，和“我改变了历史”的产品承诺冲突。放弃。

### 方案 B：动作与结果双模式

玩家每次先选“尝试行动”或“钦定结果”。自由度高，但多出一个需要解释的模式开关，移动端交互和模型协议都会变复杂。暂不采用。

### 方案 C：玩家写正史，AI 推世界

三次自由输入全部视为玩家钦定的历史结果。模型无权判断成败，只负责把结果写入时间线并推演受益者、承担者、即时后果和传播机制。该方案最符合标题和比赛展示，采用。

## 一、玩家拥有结果主权

事件页的第四条路改名为“直接改写结果”，每局仍有三次。

玩家输入的是完成时事实，例如：

> 我暗杀了皇帝，并成功让摄政会议接受了伪造的遗诏。

该句必须原样保留为新正史。DeepSeek 不得改成“试图暗杀”“暗杀失败”“只能接近皇帝”或任何降格版本。最终实现采用更严格的权限边界：客户端不信任模型返回的任何结果文本，只读取 `deviationClass`，并从玩家原文、人格和当前身份确定性构造权威回执；DeepSeek 的创造性推演从下一幕开始。

权威回执包含：

- `declaredOutcome`：玩家钦定的事实，2-80 字，不改变成败关系。
- `canonStatus`：固定为 `玩家钦定`。
- `personalityLens`：包含当前四位人格代码，说明本局人格会优先看见哪类后果，但不能改变结果。
- `causalMechanism`：结果通过命令、谣言、账本、市场、法律、迁徙等何种媒介进入社会。
- `deviationClass`：微调、改制或断裂，仅用于前端偏离度计算。
- `instantEcho`：直接结果、意外代价、受益者和承担者。

模型漏掉玩家事实时，结构校验失败并进入修复；即便模型在其他字段中夹带相反结论，reducer 也会丢弃全部模型文本并重建回执。本地保底使用同一构造器，因此任何调用路径都只能把玩家原文写入正史。

## 二、因果涟漪路由器

后续章节不能由模型自由决定继续追哪一条线。前端为第 2-11 幕选择一个 `rippleLens`，并把它作为强约束写进 prompt 和输出 schema。

可选社会载体：

- `power`：权力、军令和继承。
- `livelihood`：家庭、劳动、粮食和普通生活。
- `knowledge`：记录、教育、新闻和知识传播。
- `technology`：工具、工艺、基础设施和生产方式。
- `culture`：信仰、礼俗、语言和集体记忆。
- `trade`：价格、税收、商业和资源流动。
- `migration`：人口移动、城市和族群关系。
- `ecology`：土地、水利、疾病和环境。
- `diplomacy`：边境、联盟与外部秩序，不等于强制跨洲。

路由规则：

1. 使用历史入口、全部玩家选择、章节号生成确定性哈希。
2. 排除最近两幕已经使用的载体。
3. 优先选择与最近三幕核心冲突不同的载体。
4. 地域可以连续，但人物身份、主要受影响人群和社会场景至少更换两项。
5. 第 4 幕起，原始历史事件只能作为因果源出现一次，不得成为标题或当前任务。

每幕新增：

- `rippleLens`：必须等于前端指定载体。
- `causalBridge`：54 字以内，明确写出“玩家上次结果 -> 传播媒介 -> 当前新冲突”。

事件页的因果回执显示“蝴蝶转向”，让玩家能看懂为什么从上一幕跳到当前身份，而不是误以为游戏换了无关故事。

## 三、四选一时空人格

仍保留四次困境和最终四位代码，但每道题提供四个具体选择：同一维度的两个方向各有两种不同策略。选项显示 `A/B/C/D`，不提前显示人格字母，避免问卷感。

- 情报困境：独自核实、暗中布网、召集小联盟、公开制造压力。
- 证据困境：核实物证、小范围实验、追踪隐藏模式、押注全新可能。
- 代价困境：计算总体后果、坚持一致规则、先救具体的人、先建立共识。
- 失控困境：锁定顺序、设置阶段目标、利用突发变量、保留多条退路。

人格的作用保持克制且可见：

1. 每幕恰好一个 AI 选项按人格生成。
2. 因果预判根据人格显示不同字段。
3. 自由改写的结果永远由玩家决定；人格只决定 AI 优先揭示哪类隐藏后果。

## 四、数据与兼容

- 12 节点、50 张历史卡、三次自由改写和 v6 存档键保持不变。
- 已经处于 `custom-action` 请求的存档恢复后，按新的“结果主权”协议继续，不丢失输入。
- 旧已完成选择只保存选择文本和即时回响，不依赖旧裁决结构，因此无需升级存档版本。
- `TimelineTurn` 增加 `rippleLens` 和 `causalBridge`，旧存档幕次缺字段时由存储迁移补入确定性默认值，存档版本升级为 v7。

## 五、验收标准

- 输入“我暗杀了皇帝且成功”，输出和世界回响必须保留“成功”，不得出现失败、未遂或降格执行。
- 自由改写的 DeepSeek JSON 与本地保底都标记 `玩家钦定`。
- 连续三幕的 `rippleLens` 不重复，且 UI 显示上一次结果如何传到新社会场景。
- 第 4 幕起，开局事件不再成为标题或当前任务。
- 四道人格题每题恰好四个可点击选项，390 x 844 一屏可操作；短屏保留内部滚动。
- 真实 DeepSeek 至少完成一次自由结果改写和一次后续幕生成。
