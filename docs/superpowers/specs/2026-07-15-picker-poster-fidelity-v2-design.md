# 历史选择页一体化海报复刻设计

## 目标

以用户提供的 `codex-clipboard-d895be3b-33c6-4890-bbcb-2dd026d1ac4a.png` 为唯一视觉真相，修复当前选择页的两类主要偏差：

1. 时间轴过短、轴线与节点错位、节点文字缺少稳定的下方空间。
2. 历史卡被看成“上方大方块 + 下方小方块”，缺少参考图中一张连续高海报的整体感。

本次只调整首页胶片模式及其响应式视觉，不改 100 张历史数据、网格模式、设置菜单功能、当前节点同步、游戏逻辑、DeepSeek、存档或结局。

## 视觉结构

### 顶部与时间轴

- 保留艺术字和单一 44px 设置按钮。
- 时间区仍显示当前年份、滑动提示和 `n / 100`。
- 时间轴在时间区内水平居中，固定可视长度约为手机宽度减 64px。
- 轴线位于时间区偏上位置；所有节点圆点的圆心必须压在线上。
- 节点年份只显示在轴线下方，预留独立高度，不与轴线或卡片相撞。
- 当前节点保持在时间轴视觉中心，点击时间点与滑动卡片继续双向同步。

### 一体化高海报

- 390px 视口下主海报宽约 294px，左右保留约 48px，让相邻卡稳定露出。
- 主场景约 378px 高；朱砂年份条约 50px 宽，作为画面内部的竖向书脊，而不是独立侧栏。
- 卡序计数位于场景右下，贴近信息纸上沿。
- 信息纸与场景同宽，向上压住场景约 24px；不再缩窄成第二个小矩形。
- 信息纸取消固定大空白，使用 `min-height` 让四行内容自然决定高度。
- 标题、地点、身份、抉择、时限保持完整，不截断，不低于 11px。
- 主按钮比海报略窄，作为独立朱砂木牌置于信息纸下方，保持 68px 触控高度。
- 相邻卡暗化，仍保持相同海报轮廓。

## 资产

- 使用内置 ImageGen 生成两张无文字、无 UI 的项目资产：
  - 正面平铺、撕边、暖象牙色的空白档案纸，使用色键去背后作为动态信息纸底图。
  - 正面平铺、磨损朱砂色的空白行动木牌/纸牌，使用色键去背后作为动态主按钮底图。
- 两张资产只提供材质和不规则边缘；动态文案继续由 React 渲染。
- 现有历史事件图片、艺术字、档案舞台背景继续复用。

## 响应式与可访问性

- 390 × 844 为主验收视口，320/350/360 × 700 为防裁切视口。
- 卡宽使用 `clamp`，320px 下不得小于 252px；下一张卡始终清晰可见。
- 短屏允许选择页整体纵向滚动，主按钮必须完整可达。
- 时间点、设置按钮和主行动保持键盘可用；模式/筛选/声音行为不变。

## 验收

- 将参考图与 390 × 844 实际浏览器截图放入同一张对照图。
- 必须同时检查完整页面、时间轴局部、海报/信息纸连接局部。
- 设计 QA 无 P0/P1/P2 才能提交。
- 自动验证运行 Vitest、TypeScript、生产构建、可移植性检查和 `git diff --check`。
- 本次不再运行真实 12 节点游戏或 soak。
