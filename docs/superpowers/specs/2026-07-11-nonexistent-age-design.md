# 《不存在的时代》产品与技术设计

日期：2026-07-11  
状态：已授权自主收敛并进入实施

## 1. 产品命题

《不存在的时代》是一款移动端 AI 即兴人生模拟游戏。玩家用五次选择，活完一个由模型现场发明、从未存在过的人生与时代。

作品不使用人工题库、固定世界观或预生成剧情树。每一幕的时代规则、人物关系、冲突、选项与后果都由 DeepSeek V4 Flash 根据此前完整经历即时生成。产品只约束输出协议、五幕节奏、状态范围和内容安全边界。

核心体验承诺：

- 每次开始都是一个全新的时代、身份与问题。
- 前面选择产生的事实必须在后续被引用，并在结局兑现。
- 单局 3 至 5 分钟，任何玩家无需学习即可完成。
- 结局具有记忆点，能生成适合保存与分享的个人史册。

## 2. 赛事目标

设计直接对应互动空间的主要评审维度：

- 玩法与交互：五幕、三选一、即时回响、状态变化，形成清晰的重复游玩动机。
- 创意与惊喜：题目、制度和历史因果完全实时生成，不存在固定答案。
- 情感与叙事：以一个人的一生承载宏观时代变化，让个人选择和文明结局同时落地。
- 视觉表现：以“失落文明档案馆”为统一艺术方向，使用原创生成图像而非通用组件皮肤。
- 分享潜力：结局生成称号、稀有度、关键因果与一句可传播文案。

成功标准：首次进入 10 秒内理解玩法；一局可在 5 分钟内完成；结局明确回收至少三个此前选择；玩家愿意立即再开一局。

## 3. 核心流程

### 3.1 开始

首屏不是营销页，而是可直接操作的游戏封面。玩家点击“降生”后，模型生成作品标题、时代名称、角色身份和第一幕事件。

不提供自由文本输入，也不让玩家配置 Prompt。玩家的全部表达通过模型现场生成的三个选择完成。

### 3.2 五幕结构

1. 降生：确定身份和时代困境。
2. 生存：第一次为关系、资源或信仰付出代价。
3. 野心：出现改变阶层或制度的机会。
4. 巨变：旧选择引发战争、革命、灾难或繁荣。
5. 定史：玩家做最后决定，模型判断个人与时代的共同结局。

这五个阶段只规定叙事功能，不规定任何具体题材。模型可以自由决定年龄、时间跨度、文明形态和事件内容。

### 3.3 每幕交互

每幕显示：

- 当前幕名、角色年龄或时代时间。
- 90 至 160 字的事件正文。
- 三个语义明确、价值观不同的选择。
- 四项角色状态：权势、生计、人心、自由。
- 一条短促的时代脉搏提示。

选择后先展示“历史回响”，说明直接代价和状态变化，再生成下一幕。生成期间不使用普通转圈，而显示与世界构建有关的动态短句。

### 3.4 结局

最终生成《不存在的人生档案》，包含：

- 人物最终称号与墓志铭。
- 个人结局和文明结局。
- 五幕时间线。
- 三条关键因果链，每条必须引用真实选择。
- 稀有度名称与 0 至 100 的稀有指数。
- 一句适合分享的第一人称文案。

玩家可保存史册图片、调用系统分享能力，或点击“再活一次”重置全部世界内容。

## 4. 即兴生成模型

### 4.1 原则

系统不创建开局世界设定，也不从模板库抽题。每次请求只向模型提供：

- 固定的输出结构与内容安全规则。
- 当前是第几幕及该幕的叙事功能。
- 当前角色状态。
- 此前每幕原始事件、玩家选择和历史回响。
- 模型此前留下的因果账本。

模型可以推翻常识，但不能推翻自己已经写入历史的事实。

### 4.2 状态

四项可见状态均为 0 至 100：

- `power`：改变他人和制度的能力。
- `livelihood`：资源、财富与生存保障。
- `bonds`：信任、声望与关系网络。
- `freedom`：摆脱角色和制度束缚的程度。

客户端只负责范围校验和展示。初值、每次变化及其叙事原因均由模型生成。

因果账本是模型生成的短事实数组，每项包含已经成立的事实、未来可能付出的代价和来源幕次。它不是预设世界观，只是保证即兴叙事记得自己说过什么。

### 4.3 事件输出协议

每次事件响应必须是 JSON，并符合以下语义：

```ts
type EventTurn = {
  sessionTitle: string;
  eraName: string;
  chapter: 1 | 2 | 3 | 4 | 5;
  chapterName: string;
  timeLabel: string;
  character: {
    name: string;
    age: string;
    identity: string;
  };
  scene: {
    location: string;
    narrative: string;
    visualTone: "maritime" | "desert" | "forest" | "celestial" | "industrial" | "polar";
    worldPulse: string;
  };
  choices: Array<{
    id: "A" | "B" | "C";
    label: string;
    subtext: string;
    impulse: string;
  }>;
  stats: {
    power: number;
    livelihood: number;
    bonds: number;
    freedom: number;
  };
  immediateEcho: string | null;
  causalLedger: Array<{
    fact: string;
    debt: string;
    sourceChapter: number;
  }>;
  callbackUsed: string | null;
};
```

第一幕的 `immediateEcho` 为空。第二至第五幕必须先回响上次选择，并至少引用一项此前事实。三个选项不得只是同一动作的语气变体，也不得透露数值结果。

### 4.4 结局输出协议

```ts
type ChronicleEnding = {
  finalTitle: string;
  epitaph: string;
  personalEnding: string;
  civilizationEnding: string;
  timeline: Array<{
    chapter: number;
    timeLabel: string;
    event: string;
    choice: string;
    consequence: string;
  }>;
  causalChains: Array<{
    origin: string;
    transformation: string;
    payoff: string;
  }>;
  rarity: {
    label: string;
    score: number;
    reason: string;
  };
  shareLine: string;
};
```

结局不得增加此前从未出现、却决定胜负的关键设定。

## 5. AI 调用设计

使用 DeepSeek OpenAI 兼容接口：

- Base URL：`https://api.deepseek.com`
- Chat endpoint：`/chat/completions`
- Model：`deepseek-v4-flash`
- Thinking：enabled
- Reasoning effort：high
- Response format：JSON object
- Stream：false，优先保证结构化响应可验证

一局共六次主要请求：第一幕一次、后续四幕各一次、结局一次。每次请求重新提交经过压缩的完整游戏账本，不依赖服务端会话。

当前版本按用户明确要求采用纯前端调用。API Key 仅放在被 Git 忽略的 `.env.local`，通过 Vite 环境变量注入；这只能防止密钥进入仓库，无法防止浏览器使用者查看密钥。该边界只适用于当前本地自用版本。

## 6. 稳定性与错误处理

- 所有响应使用 Zod 校验，数值在客户端限制到 0 至 100。
- 响应含 Markdown 围栏或额外文本时，解析器只提取第一个完整 JSON 对象。
- 结构错误时自动发起一次“仅修复 JSON”请求，不改变故事内容。
- 网络错误、429 和 5xx 最多自动重试两次，采用短退避。
- 单次请求 45 秒超时；超时后保留当前局面并提供“重织这一页”。
- 请求期间锁定选择，避免重复点击造成并发分叉。
- 每完成一幕写入 `localStorage`，刷新页面可以继续本局。
- 开启新人生时终止旧请求，防止旧响应覆盖新状态。
- 安全提示词要求完全架空，避免真实政治人物、现实国家冲突、受保护 IP、色情、仇恨和未成年人不当内容。

## 7. 视觉与交互设计

### 7.1 方向

总体艺术方向是“失落文明档案馆”：不是传统古风，也不是通用暗色游戏 UI。画面将博物馆档案、未来考古、东方印刷与异文明器物结合。

主色包含墨黑、骨白、朱砂红、氧化铜绿和旧金，避免单一棕色、蓝色或紫色统治界面。卡片圆角不超过 8px，不使用渐变、装饰光球、嵌套卡片或手绘 SVG。

### 7.2 页面

- 封面：全屏原创世界图像，品牌名和“降生”按钮直接进入游戏。
- 事件页：上半部为对应 `visualTone` 的原创场景图；下半部为不加外层卡片的正文与三个选择。
- 回响层：选择后短暂覆盖场景，显示后果、状态变化和被改写的历史事实。
- 结局页：可滚动史册与独立分享画布，核心称号在首屏可见。

### 7.3 生成资产

在实施前使用 ImageGen 生成三个 390 x 844 移动端视觉方案，由执行者按赛事传播力、可读性和实现可靠性自行选定一个明确目标稿。随后生成并落地：

- 一张封面主视觉。
- 六张与 `visualTone` 对应的场景背景。
- 一张低对比纸张或织物纹理。
- 一枚作品印记和一张结局史册装饰图。

所有资产统一艺术方向，转为压缩 WebP；目标总包体积低于 8MB。图标使用 Phosphor Icons，不自绘图标。

## 8. 前端架构

技术栈：React、TypeScript、Vite、Zod、Vitest、Playwright。应用以 390 x 844 为基准移动端视口，在更宽屏幕中保持居中且不拉伸核心布局。

模块边界：

- `src/game/types.ts`：领域类型。
- `src/game/schema.ts`：模型响应校验。
- `src/game/prompts.ts`：系统提示词、事件请求和修复请求。
- `src/game/reducer.ts`：局内状态机。
- `src/services/deepseek.ts`：唯一 API 边界、重试和超时。
- `src/services/storage.ts`：局内恢复。
- `src/screens/*`：封面、事件、回响、结局与错误状态。
- `src/components/*`：状态条、选择、进度和分享史册。

状态机阶段为 `idle -> generating -> event -> echo -> generating -> ending -> chronicle`，错误状态保留最近一次有效阶段。

## 9. 分享设计

结局页可生成 1080 x 1440 的史册图片，包含作品名、人物称号、时代名、稀有度、三条因果摘要和重新游玩的提示。优先调用 Web Share API；不支持时保存图片并复制分享文案。

分享文案只描述结果，不使用“分享后解锁”等诱导传播机制。

## 10. 测试与验收

单元测试覆盖：

- 合法与非法 AI 响应解析。
- JSON 提取、数值限制和修复请求。
- 五幕状态机与重复点击保护。
- 提示词包含此前选择和因果账本。
- 本地存储恢复和版本不兼容清理。

端到端测试使用固定的模拟 AI 响应覆盖：

- 从降生到史册的完整五幕流程。
- 选择后的回响和状态变化。
- 刷新恢复、错误重试和重新开始。
- 保存或分享结局。

真实 API 验收至少连续完成三局，确认：

- 每局题目和世界明显不同。
- 每幕三个选项有效且不重复。
- 后续至少三次明确引用此前选择。
- 结局不引入决定性的新设定。
- 390 x 844 无横向滚动、遮挡、截断或离屏主操作。

视觉验收使用选定 ImageGen 目标稿与同视口截图逐项比较。`design-qa.md` 必须记录并修复所有 P0、P1、P2 问题，最终结果为 `passed` 后才能交付。

## 11. 本期不做

- 账号、云存档、排行榜和多人玩法。
- 玩家自由输入 Prompt。
- 预设历史世界、人工题库或固定剧情分支。
- 运行时图像生成。
- 付费、广告、社交关系链和后台管理。
- 将当前前端密钥方案视为可公开上线的安全架构。

## 12. 决策记录

- 选择全程即兴叙事，接受比预生成世界更高的一致性风险。
- 用历史记录和因果账本缓解漂移，但不预设世界规则。
- 采用五幕短局制，优先保证游园会现场可完成与可重玩。
- 当前仅做纯前端本地版本，直接调用 DeepSeek V4 Flash。
- 用户授权后续设计与实现节点由执行者自主判断，不再逐项咨询。
