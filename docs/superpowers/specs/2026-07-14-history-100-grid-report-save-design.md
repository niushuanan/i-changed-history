# 百节点、双浏览与结局保存设计

## 目标

本次版本在不改变“固定首幕、同一主角十二次决定、DeepSeek 只负责后续世界推演”的前提下完成四项产品升级：历史入口从 50 扩为 100（中国 60、世界 40）；新增与时间胶片共享当前节点的两列网格；重新排布“被改变的 2026”；让两份最终报告都能导出完整高清图片，并在移动端进入系统保存流程、桌面端直接下载。

稳定优先级如下：历史卡必须真实、著名、有明确现场决定；模式切换不能丢失当前节点；搜索筛选不进入游戏状态机；报告不裁字；保存失败不影响已经完成的游戏。

## 方案比较与选择

### 方案一：在现有胶片卡组件中增加紧凑分支

改动文件少，但一个组件同时承担 560px 胶片卡与两列网格卡，条件样式、图片尺寸、点击区域和无障碍语义会互相牵制。100 张卡同时隐藏挂载还会造成重复按钮与无谓图片负担。

### 方案二：共享目录状态，胶片与网格各自渲染（采用）

`SeedPickerScreen` 只维护一个稳定的 `activeSeedId` 和一组浏览条件；模式 A 继续使用完整胶片卡，模式 B 使用独立紧凑网格卡；纯函数目录模块负责搜索、筛选、主题映射和年代格式。切换只更换渲染器，不复制数据、不进入 reducer。

### 方案三：网格作为独立路由

职责清晰，但会引入路由、返回路径和跨页面状态恢复，用户需要多一步，并与“一键切换且保持当前节点”相悖。

## 一、百节点历史目录

保留现有 50 张，新增 30 张中国与 20 张世界卡，最终精确为 60/40。新增卡覆盖公元前、古代、中世纪、近世、工业化和现代，不用冷门小国事件填数。中国新增重点补足先秦、秦汉和 1898—1978 的重大转折；世界新增重点补足公元前、制度、医学、工业、殖民地独立、能源与冷战终结。

每张卡仍必须包含：稳定 id、精确年与日期标签、地点、被分配的具体角色、一个立即决定、明确截止时间、实际历史结果、三条史实锚点、领域与视觉语调。公元前年份使用负数存储，但所有界面和模型上下文统一格式化为“公元前 N 年”，不能显示负号年份。

首幕继续由本地同步生成并过现有 schema；新增 50 张必须逐张通过固定首幕解析，不能触发模型请求。100 张卡按真实年代排序，同年再按稳定目录顺序排序。

### 图片

每个新增 id 必须拥有独立的 `/assets/history/<id>.webp`。抓取工具按条目分批调用 Wikipedia/Commons API，下载本地缩略图并转换为 WebP；同时保存原始页面、文件页、作者、许可证、许可证链接和是否使用本地年代画作兜底。无法确认复用条件或找不到事件相关图时，宁可使用项目已有年代画作并明确标注“时代氛围图”，不使用运行时远程链接。

## 二、双浏览与筛选

默认仍为模式 A“时间胶片”。标题区增加“胶片 / 网格”双段切换，不增加弹窗或第二页面。当前节点由 `activeSeedId` 表示：

- A → B：网格滚到同一节点并用朱砂状态标识。
- B → A：时间轴与胶片都回到同一节点。
- 进入游戏再退出：恢复此前模式、筛选条件和节点。
- 筛选暂时隐藏当前节点时，不擅自改写当前节点；清除条件后仍恢复它。

模式 B 顶部包含搜索、时间范围、地域、属性主题、结果数与清除入口。搜索覆盖事件名、年份、地点、角色、领域、结果和三条史实；维度之间执行 AND。时间范围使用“全部 / 公元前 / 公元 500 年前 / 500—1499 / 1500—1899 / 1900 以后”；地域使用“全部 / 中国 / 世界”；主题使用“全部 / 军事 / 政治制度 / 经济社会 / 科学技术 / 文化交流”。

网格在 390px 宽度保持每行两张，整卡是 44px 以上的可操作目标，展示图片、年份、事件名、地点和主题。筛选为空时保留条件，显示结果为空原因和“清除筛选”。

## 三、2026 报告排版

“2026，普通人的一天”将三条数据按原有顺序合并为一个 `<p>`，用中文全角分号自然连接，不修改模型原文。它在常见桌面报告宽度中目标约两行，在 390px 移动端允许自然换行，不能以裁字、行夹或省略号强行达到两行。

紧随其后的小说式结尾与分享句获得释放出的高度：结尾正文、注释/分享句明显放大。四时代列表取消拉伸填满剩余高度的 `1fr` 行，改为内容驱动的紧凑行；减小无意义列宽和内边距，并把时代、标题、正文、继承变化及周边文字整体提高到至少 11px。报告区允许滚动，所有 AI 文案完整保留。

“穿越者列传”同步删除依赖后置覆盖才取消的旧截断规则，避免两套相反 CSS 继续共存。

## 四、完整高清保存

导出对象是当前激活的单份报告。生成时读取完整 `scrollWidth` 和 `scrollHeight`，克隆节点取消滚动裁切，以 2 倍像素比生成 PNG；切换报告页会使已经准备的旧图片失效，防止保存错页。

桌面端点击“保存这一页”后生成并自动下载 PNG，文件名区分“穿越者列传”或“被改变的 2026”。移动端采用两阶段流程：

1. 第一次点击生成完整高清图片，并显示“图片已准备好”。
2. 用户再次点击“打开系统保存”时，以新的用户手势调用 Web Share 文件能力；界面明确提示在系统面板选择“存储图像/保存到相册”。
3. 若浏览器不支持文件分享或系统面板没有保存入口，始终提供“下载 PNG”兜底。

移动 Web 不伪装不存在的相册写权限，也不把“打开分享面板”宣称为“已经写入相册”。状态文案必须区分已下载、已打开系统面板、已取消和生成失败。

## 错误与恢复

- 图片抓取失败：记录兜底原因，使用本地年代图；运行时绝不依赖远程图片。
- 搜索筛选为空：不重置条件，不改变当前节点。
- PNG 生成失败：保留结局与当前报告页，允许重试。
- 移动分享取消：回到“图片已准备好”，不触发下载。
- 移动分享异常：保留手动下载入口，不丢弃已生成文件。

## 验收

自动化测试必须证明：100/60/40 与 id 唯一；新增 50 张图片和授权清单存在；所有 100 张固定首幕可解析；公元前年份无负号显示；筛选组合、空结果、A/B 上下文保持、退出恢复；普通人的一天只有一个自然段；四时代列表无强制空白；完整滚动尺寸进入 PNG 渲染；移动分享与桌面下载状态区分明确。

浏览器验收使用 390×844 与常见桌面宽度：模式 A 首屏仍露出下一张卡；模式 B 每行两张、可上下滚动；切换保持节点；筛选流畅；报告无 7—9px 正文、无 clamp/ellipsis；两份报告可滚到结尾；导出图片包含当前报告最后一句。

## 不做

不引入人格系统、推荐节点、随机洗牌、档案弹窗、第二套时间线、远程运行时图片、原生壳、服务端存储或新的模型请求。搜索和筛选只服务百节点入口，不参与 AI 的剧情选择。
