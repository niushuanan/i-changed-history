# 《哎！我改变了历史？》前情与历史对照设计

## 目标

事件页必须在一个手机屏幕内让玩家读懂三件事：上一项决定如何把自己带到此刻、眼前真实人物正在争夺什么、这条新历史与真实历史究竟哪里不同。完整文字优先于大图，任何 AI 正文都不能用省略号截断。

## 前情叙事

- 沿用现有 `narrative` 字段，不新增接口、不拆分请求，避免降低生成稳定性。
- 正文从最多 56 字扩展到最多 88 字，并要求模型恰好写两句。
- 续幕第一句说明上一项决定如何造成当前局面，第二句交代真实人物的意图、可见物件和迫近冲突。
- 第一幕第一句交代著名历史前情，第二句交代玩家此刻必须处理的冲突。
- 正文与期限都完整换行显示，不使用 `line-clamp`、`text-overflow: ellipsis` 或单行裁切。

## 历史对照

底部保留现有紧凑证明区，但固定拆成三层：

1. `被你改变后`：显示当前架空历史已经成立的新事实，使用 `worldStateChange`。
2. `真实历史中`：只显示真实历史实际发生的结果和直接后果，使用 `divergenceProof`。
3. `变化来自`：显示玩家上一项决定与新结果之间的因果桥，使用 `lastChoiceLabel` 和 `causalBridge`。

这三个字段仍由一次 DeepSeek 回合请求共同生成，因此共享同一份人物、地点、年代与完整决策正史。前端只负责清晰对照，不在本地杜撰历史。真实历史字段必须包含至少一个可核验的人物、机构、地点或事件锚点；架空结果与真实结果不得混写。

## 自适应排版

- 事件页保持连续电影画面，不增加卡片、弹窗或指标面板。
- 文案较长时缩短场景图，把高度让给完整前情、三项行动和历史对照。
- 对照区按内容自然增高，取消垂直居中造成的空洞感。
- 三个行动仍是页面主要交互，位置保持在历史对照之前。
- 在 390 x 844 下不得页面滚动、不得文字覆盖、不得大块无意义空白。

## 验收

- 顶部前情能完整说明从上一决定到当前冲突的因果过程。
- 底部清晰显示 `被你改变后`、`真实历史中`、`变化来自`。
- 页面中由 CSS 制造的省略号全部消失，AI 正文完整可见。
- 不新增模型请求或响应字段，生成速度和失败面不劣化。
- 单元测试、类型检查、构建、390 x 844 真实浏览器截图和点击验证通过。
